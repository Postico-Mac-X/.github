## 💡 Overview

<a href="#postico">Postico</a> is a native Mac PostgreSQL database client — providing a beautiful, intuitive interface for connecting to, querying, and managing PostgreSQL databases that reflects the design quality of the best Mac applications rather than the functional-but-ugly aesthetic common to cross-platform database clients.

The <a href="#postico">table content view</a> is Postico's primary interface — displaying database table contents in a spreadsheet-like grid that makes data immediately readable, browsable, and editable. Rows display as table rows, columns as table columns, and null values, data types, and long text content are handled intelligently for readable display. This direct visual engagement with table data is how most database work actually happens — browsing data to understand its structure, finding specific records, checking that operations produced the expected results. The <a href="#postico">row editing capability</a> modifies table data directly from the content view — clicking into a cell to edit its value, inserting new rows, deleting rows, and committing changes to the database. Editing table data doesn't require writing UPDATE or INSERT SQL for simple operations — the visual editing interface handles these directly.

The <a href="#postico">SQL query editor</a> provides a full-featured environment for writing and executing SQL queries — syntax highlighting for PostgreSQL SQL, query execution with keyboard shortcuts, query results displayed in a scrollable grid, and the ability to save and recall frequent queries. The editor supports the complete PostgreSQL SQL feature set for complex queries, stored procedures, and database administration operations. The <a href="#postico">database structure browser</a> navigates the complete database schema — tables, views, functions, sequences, indexes, and other database objects — providing visibility into how the database is structured without needing to write information_schema queries manually.

<a href="#postico">Connection favorites</a> save database connection profiles — host, port, database name, username, and SSL configuration — for instant reconnection to frequently used databases without re-entering connection details. Multiple connection favorites enable switching between development, staging, and production databases quickly. <a href="#postico">Filter and sort in content view</a> narrows table display by column values and sorts rows by any column — finding specific records and navigating large tables without writing WHERE and ORDER BY SQL.

<a href="#postico">Export query results</a> saves the results of any query to CSV or other formats — extracting data for analysis, sharing with non-technical stakeholders, and importing into other applications that accept tabular data. <a href="#postico">PostgreSQL-specific features</a> include support for PostgreSQL data types — arrays, JSON, UUID, geometric types, and other PostgreSQL-specific column types displayed and editable appropriately in the table view.

## ❓ FAQ

<details>
<summary>Does Postico support only PostgreSQL?</summary>

Yes. Postico is specifically designed for PostgreSQL databases. It does not support MySQL, SQLite, or other database systems.

</details>

<details>
<summary>Can I edit data directly in Postico's table view?</summary>

Yes. Row editing allows clicking into cells to modify values, inserting new rows, and deleting rows directly from the table content view without writing SQL.

</details>

<details>
<summary>Does Postico have a SQL query editor?</summary>

Yes. A full-featured SQL editor with syntax highlighting, keyboard shortcut execution, and scrollable results handles complex queries and administrative operations.

</details>

<details>
<summary>Can Postico save database connection details?</summary>

Yes. Connection favorites save complete connection profiles including host, port, database name, username, and SSL configuration for instant reconnection.

</details>

<details>
<summary>Does Postico support SSH tunnels?</summary>

Yes. SSH tunnel connections enable access to remote databases not directly accessible on the network.

</details>

<details>
<summary>Can I export query results from Postico?</summary>

Yes. Query results export to CSV and other formats for use in analysis tools and sharing with others.

</details>

<details>
<summary>Does Postico support dark mode?</summary>

Yes. Postico follows the macOS system appearance setting for dark mode support.

</details>

<details>
<summary>Does Postico support PostgreSQL-specific data types?</summary>

Yes. Arrays, JSON, UUID, geometric types, and other PostgreSQL-specific column types display and are editable appropriately in the table view.

</details>

<details>
<summary>Can I have multiple database connections open in Postico?</summary>

Yes. Multiple simultaneous connections open in separate windows for working with different database environments concurrently.

</details>
